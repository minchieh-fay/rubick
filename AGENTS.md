## 项目背景

Rubick 是一个带后端服务的客户端 work app，形态上接近协同工作台类产品，并保留未来演进为桌面 App 的空间。

它不是纯前端项目，包含：

- `server/`：服务端
- `client/web/`：客户端页面
- `client/server/`：客户端本地后台
- `shared/`：共享类型与工具

## 必须遵守

1. 任何框架性、目录结构、设计文档相关变更，都必须同步更新 `docs/` 下相关文档。
2. 不要把本项目当成单纯网页项目来改。
3. 客户端和服务端都要遵守模块独立性规则。
4. 一个 HTTP 业务模块尽量放在一个独立目录中。
5. 主业务逻辑放主业务文件；不需要重点阅读的辅助函数、辅助方法和辅助类逻辑放模块自己的 `help.ts`。
6. 服务端模块的 `.env` 和 sqlite 文件应尽量留在模块目录内。
7. 文档路径请使用相对路径或普通仓库路径，不要写死本机绝对路径。
