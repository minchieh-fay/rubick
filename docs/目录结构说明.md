# 目录结构说明

## 总览

```
rubick/
├── core/                  # 核心基础设施
│   ├── cli/              # CLI 工具抽象层
│   │   ├── index.ts      # 导出
│   │   ├── types.ts      # ToolDefinition, ToolResult 接口
│   │   └── wrapper.ts    # CLI 工具创建器
│   ├── loader/           # 插件加载与注册
│   │   ├── index.ts      # 导出
│   │   ├── registry.ts   # ToolRegistry 注册表
│   │   └── loader.ts     # 目录扫描与动态加载
│   └── agent/            # Agent 运行时
│       ├── index.ts      # 导出
│       └── types.ts      # AgentConfig, AgentRuntime
│
├── builtin/               # 内置插件（官方维护，Git 追踪）
│   ├── tools/            # 内置工具
│   │   ├── file-reader/  # 示例：文件读取工具
│   │   └── git/          # 示例：Git 命令封装
│   └── skills/           # 内置技能
│       └── requirement-analysis/  # 需求分析技能
│
├── custom/                # 用户自定义插件（本地，不提交 Git）
│   ├── tools/            # 用户自定义工具
│   └── skills/           # 用户自定义技能
│
├── server/                # HTTP 服务（开发调试）
│   └── index.ts          # Hono 服务入口
│
├── client/                # 客户端
│   └── web/              # Web 界面
│
├── electron/              # Desktop App 打包
│
├── docs/                  # 项目文档
│
├── package.json           # 项目配置
├── tsconfig.json          # TypeScript 配置
├── .gitignore             # Git 忽略规则
├── AGENTS.md              # AI 角色定义与进度
└── README.md              # 项目说明
```

## 各目录职责

### core/

核心基础设施，提供：
- **CLI 抽象层**：统一的工具接口，上层不关心底层实现
- **加载器**：扫描目录，动态加载插件
- **注册表**：管理已注册的工具

### builtin/

官方维护的插件集合：
- 随项目分发
- 可以提交到 Git
- 保证质量和稳定性
- 每个插件独立目录，包含 `index.ts`

### custom/

用户自定义插件：
- 本地扩展，不提交 Git
- 满足个性化需求
- 结构与 builtin 相同

### server/

开发调试用的 HTTP 服务：
- 使用 Hono 框架
- 提供工具调用 API
- B/S 架构开发调试

### client/web/

Web 客户端界面：
- 用户交互界面
- 与 server 通信

### electron/

Desktop App 打包：
- Electron 主进程
- BrowserWindow 包裹 Web 客户端

## 插件结构规范

### Tool 插件

```
tools/my-tool/
├── index.ts      # 必需：导出 ToolDefinition
├── help.ts       # 可选：辅助函数
└── README.md     # 可选：使用说明
```

### Skill 插件

```
skills/my-skill/
├── SKILL.md      # 必需：技能描述和指令
├── help.ts       # 可选：辅助函数
└── ...           # 其他支持文件
```

## 路径别名

在 tsconfig.json 中配置：
- `@/*` → 根目录
- `@builtin/*` → builtin/
- `@custom/*` → custom/
- `@core/*` → core/
