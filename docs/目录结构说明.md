# 目录结构说明

## 根目录

```
rubick/
├── AGENTS.md              # AI 角色定义、协作规则、当前进度
├── docs/                  # 项目文档（产品、规范、设计）
│   ├── products/          #   产品定义文档
│   │   ├── architecture.md   #   总体架构、术语、流程
│   │   ├── hub.md            #   rubickhub 详细规格
│   │   ├── tool.md           #   rubicktool 详细规格
│   │   ├── rubick.md         #   rubick 运行端详细规格
│   │   └── specs.md          #   统一规范（包名、版本、UI、代码组织）
│   └── 其他按需补充的文档
├── memory/                # AI 记忆（固化信息、上下文、决策）
│   ├── memory.md          #   长期记忆（用户偏好、项目关键信息）
│   ├── context.md         #   当前上下文（进行中的任务、待办）
│   └── decisions.md       #   决策记录（为什么这么做）
├── yy/                    # 用户原始想法（不成熟需求，保留不删除）
└── 各工程目录（后续创建）
```

## 文档更新规则

**当以下情况发生时，必须同步更新对应文档：**

1. 代码结构变更 → 更新本文件（目录结构说明）
2. 产品逻辑变更 → 更新 `docs/products/` 下对应文档
3. 规范调整（包名、版本、UI）→ 更新 `docs/products/specs.md`
4. 重要决策 → 追加到 `memory/decisions.md`
5. 用户偏好/关键信息 → 更新 `memory/memory.md`

**原则：**
- 文档是活的，不是写完就扔
- 改代码的同时改文档，不让文档和代码脱节
- yy/ 下的原始文件保留，不删除
