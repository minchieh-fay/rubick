# Rubick 目录结构说明

本文档用于说明当前工程的第一期基础骨架，目标是先把服务端、客户端和共享层的职责拆清楚，便于后续逐步扩展。

相关文档：

## 设计原则

1. 服务端与客户端分离，避免后期耦合过深。
2. 服务端按模块组织，每个模块一个独立文件夹，模块目录自己维护 `.env`，如果业务需要数据库，也在自己的目录内生成 sqlite 文件。
3. 客户端默认按 B/S 架构推进，同时保留本地 Bun 后台，方便未来切换到 Electrobun 打包为桌面 App。
4. 公共类型和工具独立存放，避免重复定义。

## 各目录职责

### `server/`

服务端工程目录，后续主要负责模型适配、统一接口暴露、模块自动发现与加载。

- `src/main.ts`
  服务端主入口。后续用于启动 HTTP 服务、初始化配置、装载模块。
- `src/bootstrap/`
  放启动期逻辑，例如扫描模块目录、加载配置、初始化数据库连接等。
- `src/core/`
  放服务端核心抽象，例如模块接口、生命周期定义、基础上下文对象等。
- `src/http/`
  放统一 HTTP 层能力，例如路由注册、中间件、请求上下文处理。
- `src/modules/`
  放业务模块。每个模块一个独立目录，模块自己的 `.env`、sqlite 文件和业务实现都放在自己的目录里。
- `src/utils/`
  放服务端内部通用工具函数。

### `server/src/modules/example/`

示例模块目录，用于约定今后每个模块的大致形态。

- `index.ts`
  模块入口文件。既可以被主工程动态加载，也可以通过 `import.meta.main` 方式独立运行。
- `.env`
  模块自己的环境配置文件。
- `*.sqlite`
  如果模块业务需要数据库，就直接在该模块目录下生成自己的 sqlite 文件，而不是放到公共目录。
- `services/`
  放模块内部服务逻辑，例如 ASR、TTS、Chat、Embedding 的具体适配实现。

### `client/web/`

客户端前端页面目录。当前先按浏览器访问方式设计，后续打包 App 时仍可复用这部分页面层代码。

- `index.html`
  Web 页面入口。
- `src/pages/`
  页面级视图。
- `src/components/`
  通用组件。
- `src/stores/`
  前端状态管理。
- `src/api/`
  统一封装接口调用，后续可以拆成本地 API 和远端 API 两层。
- `src/assets/`
  静态资源。
- `src/main.ts`
  前端启动入口。

### `client/server/`

客户端 Bun 后台。负责监听 localhost 端口，为前端页面提供本地 API，并协调与远端服务端的交互。

- `src/main.ts`
  本地后台启动入口。
- `src/routes/`
  本地 API 路由定义。
- `src/services/`
  本地后台服务逻辑，例如配置读写、任务调度、本地缓存管理。
- `src/config/`
  本地客户端配置文件目录。
- `src/data/`
  本地客户端 sqlite、缓存、会话数据等。

### `shared/`

服务端与客户端共享的类型和工具目录。

- `types/`
  放公共 TypeScript 类型定义。
- `utils/`
  放通用工具函数。

### AI 协作文件

为了适配不同 AI 编码工具，当前工程在根目录和隐藏配置目录中增加了统一规则入口。

- `AGENTS.md`
  通用 Agent 协作说明入口，可供支持 `AGENTS.md` 的工具读取。
- `CLAUDE.md`
  Claude Code 项目规则入口。
- `.cursor/rules/project.mdc`
  Cursor 项目规则文件。
- `.trae/rules/project_rules.md`
  Trae 项目规则文件。
- `.qoder/rules/project_rules.md`
  Qoder 项目规则文件。
- `docs/AI协作规范.md`
  统一规则源文档。上述工具文件应与它保持一致。

## 后续建议

1. 在根目录补充 `package.json` 或按 server/client 分别维护 Bun 工程配置。
2. 为服务端补一个统一的模块注册协议，例如 `registerRoutes()`、`registerTasks()`、`healthCheck()`。
3. 为客户端前端选定框架，例如原生 TS、React 或 Vue。
4. 在 `shared/types/` 内先定义服务端响应结构、模块清单结构和配置结构。
5. 后面准备打包 App 时，可在 `client/` 下新增 `desktop/` 或 `electrobun/` 目录承载桌面壳。
