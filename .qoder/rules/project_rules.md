# Rubick Qoder Project Rules

请把 `docs/AI协作规范.md` 作为本项目的统一规则源，并在开始实现前阅读：

- `docs/目录结构说明.md`
- `docs/总体架构说明.md`
- `docs/编码约定.md`
- `docs/模块开发规范.md`
- `docs/通信与调用说明.md`

## 项目背景

Rubick 是一个带后端服务的客户端 work app，未来可继续演进为桌面 App。

项目不是纯前端结构，包含：

- `server/`
- `client/web/`
- `client/server/`
- `shared/`

## 必须遵守

1. 不要把本项目按单纯前端项目处理。
2. 一个 HTTP 业务域尽量放在一个独立目录中。
3. 模块优先保持自包含，尽量少依赖模块外部文件。
4. 主业务逻辑放主业务文件。
5. 不需要重点阅读的辅助函数、辅助方法和成员相关辅助逻辑放到模块自己的 `help.ts`。
6. 服务端模块自己的 `.env` 和 sqlite 文件应尽量保留在模块目录里。
7. 文档中的路径请使用相对路径。

## 文档同步要求

如果修改涉及框架、目录、设计、模块规范或通信关系，必须在同一次任务里同步更新文档。

至少检查：

- `README.md`
- `docs/目录结构说明.md`
- `docs/总体架构说明.md`
- `docs/编码约定.md`
- `docs/模块开发规范.md`
- `docs/通信与调用说明.md`
- `docs/AI协作规范.md`
