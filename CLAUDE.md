# Rubick Claude Code Rules

在本仓库工作时，请把 `docs/AI协作规范.md` 视为统一规则源，并结合以下文档一起理解项目：

- `docs/目录结构说明.md`
- `docs/总体架构说明.md`
- `docs/编码约定.md`
- `docs/模块开发规范.md`
- `docs/通信与调用说明.md`

## 项目认知

Rubick 是一个带后端服务的客户端 work app，不是纯前端项目。

核心组成：

- `server/`：服务端能力层
- `client/web/`：客户端页面层
- `client/server/`：客户端本地后台层
- `shared/`：共享协议、类型、工具

## 关键规则

1. 如果任务涉及架构、目录、规范、调用关系变化，必须同步更新相关文档。
2. 修改设计或目录时，不要只改代码不改文档。
3. 客户端和服务端都遵守模块独立性优先的组织方式。
4. 一个 HTTP 业务域尽量放在一个独立目录中。
5. 主业务代码留在主业务文件；辅助函数、辅助类方法、成员相关辅助逻辑统一收敛到模块自己的 `help.ts`。
6. 服务端模块自己的 `.env` 和 sqlite 文件应尽量留在模块目录内。
7. 文档里使用相对路径，不要写死绝对路径。
